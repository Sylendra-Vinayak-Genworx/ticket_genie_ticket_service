## What This Report Covers

This report documents the complete behaviour of the TicketGenie automatic
ticket assignment engine. Every scenario below answers: **which agent gets

### Scoring Formula

```
score = proficiency_weight + (min(tickets_resolved, EXPERIENCE_CAP) × 5) - (active_tickets × 10)

proficiency_weight:  expert=100  |  intermediate=50  |  beginner=10
experience weight :  +5 per resolved ticket in the same area (capped)
workload penalty  :  -10 per currently active ticket
workload hard cap :  agents at or above MAX_AGENT_WORKLOAD are excluded entirely
```

### Assignment Pipeline

```
New ticket arrives
      ↓
Acquire atomic DB lock (prevents race conditions)
      ↓
Area of concern known?
  YES → Skill-based scoring → pick highest score within workload cap
  NO  → Skip to least-loaded
      ↓
Skilled agents found and eligible?
  YES → Assign winner
  NO  → Least-loaded agent globally
      ↓
No agents at all?
  → Route to lead's team queue (agents self-claim)
      ↓
Lead timeout watchdog (every 2 min)
  → Move unclaimed tickets to OPEN queue
```
